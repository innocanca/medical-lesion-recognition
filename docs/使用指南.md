# 医疗病灶识别系统 - 使用指南

本文档介绍如何训练模型以及如何调用 API 接口。

---

## 一、模型训练

### 1.1 环境准备

```bash
cd medical-lesion-recognition
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

pip install -r requirements.txt
pip install -r requirements-train.txt
```

### 1.2 准备标注数据

#### 目录结构

```
data/labeled/
├── images/           # 放置病灶图像
│   ├── lesion_001.jpg
│   ├── lesion_002.jpg
│   └── ...
└── labels.jsonl      # 标注文件
```

#### 标注格式 (labels.jsonl)

每行一条 JSON，字段说明如下：

| 字段 | 必填 | 说明 |
|------|------|------|
| image_path | 是 | 相对于 `data/labeled/` 的图像路径 |
| color | 是 | 颜色，如：红斑、色素沉着 |
| size | 是 | 大小，如：米粒大小 (3-5mm) |
| position | 是 | 位置，如：躯干、面部 |
| shape | 是 | 形状，如：斑片状、圆形 |
| scale | 是 | 鳞屑程度，如：轻度鳞屑、无鳞屑 |
| texture | 否 | 质地，如：粗糙、光滑 |
| border | 否 | 边界，如：边界清楚、边界模糊 |

示例：

```json
{"image_path": "images/lesion_001.jpg", "color": "红斑", "size": "米粒大小 (3-5mm)", "position": "躯干", "shape": "斑片状", "scale": "轻度鳞屑", "texture": "粗糙", "border": "边界清楚"}
{"image_path": "images/lesion_002.jpg", "color": "色素沉着", "size": "黄豆大小 (5-10mm)", "position": "面部", "shape": "圆形", "scale": "无鳞屑", "texture": "光滑", "border": "边界清楚"}
```

术语需与 `config/config.yaml` 中 `terminology` 保持一致。

#### 快速生成测试数据（可选）

若暂无可标注数据，可生成 dummy 数据验证流程：

```bash
python scripts/prepare_dummy_data.py
```

### 1.3 执行训练

```bash
python training/train.py \
  --data-dir data/labeled \
  --epochs 20 \
  --batch-size 16 \
  --lr 1e-4 \
  --val-ratio 0.2 \
  --device cuda
```

#### 训练参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| --data-dir | data/labeled | 标注数据目录 |
| --output-dir | models/checkpoints | 模型输出目录 |
| --config | config/config.yaml | 配置文件路径 |
| --epochs | 20 | 训练轮数 |
| --batch-size | 16 | 批次大小 |
| --lr | 1e-4 | 学习率 |
| --val-ratio | 0.2 | 验证集比例 (0~1) |
| --device | cuda/cpu | 训练设备 |
| --seed | 42 | 随机种子 |

### 1.4 训练完成后

最佳模型保存至 `models/checkpoints/best_model.pt`。重启 API 服务后会自动加载该模型。

## 二、API 调用

### 2.1 启动服务

```bash
# 方式一：直接运行
python -m uvicorn api.main:app --host 0.0.0.0 --port 8000

# 方式二：Docker
docker-compose up -d
```

服务启动后：
- API 地址：`http://localhost:8000`
- 交互文档：`http://localhost:8000/docs`
- OpenAPI：`http://localhost:8000/openapi.json`

### 2.2 接口列表

#### 1) 健康检查 `GET /health`

检查服务状态及模型是否已加载。

**请求示例：**

```bash
curl http://localhost:8000/health
```

**响应示例：**

```json
{
  "status": "ok",
  "model_loaded": true
}
```

---

#### 2) 图像分析（文件上传）`POST /analyze`

上传病灶图像，返回医学规范术语描述。

**请求示例：**

```bash
curl -X POST \
  -F "image=@/path/to/lesion.jpg" \
  http://localhost:8000/analyze
```

**请求说明：**
- Content-Type: `multipart/form-data`
- 参数 `image`：图片文件（支持 JPEG、PNG）

**响应示例：**

```json
{
  "success": true,
  "lesion_description": {
    "color": "红斑",
    "size": "米粒大小 (3-5mm)",
    "position": "躯干",
    "shape": "斑片状",
    "scale": "轻度鳞屑",
    "texture": "粗糙",
    "border": "边界清楚",
    "confidence": 0.8523,
    "summary": "红斑，米粒大小 (3-5mm)，躯干，斑片状，轻度鳞屑"
  },
  "model_version": "v1"
}
```

---

#### 3) 图像分析（Base64）`POST /analyze/base64`

通过 Base64 编码的图像数据进行分析，适用于前端或无法直接传文件的场景。

**请求示例：**

```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -d '{"image": "BASE64_ENCODED_IMAGE_STRING"}' \
  http://localhost:8000/analyze/base64
```

**请求 Body：**

```json
{
  "image": "iVBORw0KGgoAAAANSUhEUgAAA..."
}
```

**响应格式：** 与 `POST /analyze` 相同。

### 2.3 调用示例

#### Python 示例

```python
import requests

# 方式一：文件上传
with open("lesion.jpg", "rb") as f:
    resp = requests.post(
        "http://localhost:8000/analyze",
        files={"image": f},
    )
print(resp.json())

# 方式二：Base64
import base64
with open("lesion.jpg", "rb") as f:
    b64 = base64.b64encode(f.read()).decode()
resp = requests.post(
    "http://localhost:8000/analyze/base64",
    json={"image": b64},
)
print(resp.json())
```

#### JavaScript 示例

```javascript
// 使用 fetch 上传文件
const formData = new FormData();
formData.append('image', fileInput.files[0]);

fetch('http://localhost:8000/analyze', {
  method: 'POST',
  body: formData
})
  .then(res => res.json())
  .then(data => console.log(data));
```

#### cURL 示例

```bash
# 健康检查
curl http://localhost:8000/health

# 图像分析
curl -X POST -F "image=@lesion.jpg" http://localhost:8000/analyze
```

### 2.4 错误处理

| HTTP 状态码 | 说明 |
|-------------|------|
| 400 | 请求参数错误（如非图片、Base64 无效） |
| 500 | 服务端内部错误 |

**400 示例：**

```json
{
  "detail": "请上传图片文件 (JPEG/PNG)"
}
```

## 三、附录

### 医学术语配置

在 `config/config.yaml` 的 `terminology` 中可修改或扩展术语，需与标注数据一致。当前支持维度：

- **colors**：颜色
- **size_categories**：大小
- **positions**：位置
- **shapes**：形状
- **scale_presence**：鳞屑
- **textures**：质地
- **borders**：边界

### 无模型时的行为

未训练模型或模型未加载时，API 仍可正常调用，会返回占位结果（`confidence=0`），便于联调与集成。
